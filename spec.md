# 故래소 — 기획 설계서

> "가장 좋은 투자 조언자는 AI가 아니라, 감정이 없을 때의 나일 수 있습니다."

**해커톤:** SKYSH 2026  
**주제:** Theme 1 — Emotion (기술로 공포와 탐욕 다루기)  
**날짜:** 2026-06-28

---

## 한 줄 설명

故래소는 증권사 API 주문 직전에 사용자의 투자 유언장을 읽어주고, 거래가 끝난 뒤 결과를 다시 유언장 조항으로 남기는 거래 실행형 투자 복기 서비스다.

---

## 최종 서비스 구조

| 구분 | 역할 | 설명 |
|------|------|------|
| **증권사 API** | 실제 주문 실행 | 매수/매도, 체결 조회, 잔고 조회, 주문 내역 조회 |
| **웹서비스** | 거래 전후 판단 보조 | 주문 전 유언장 낭독, 거래 후 사망진단서/생존보고서 발급 |
| **투자 유언장** | 개인 투자 원칙 | 과거 거래에서 얻은 교훈을 조항으로 저장 |
| **거래 보고서** | 복기 자료 | 손실이면 사망진단서, 수익이면 생존보고서 |
| **주문 전 개입 모달** | 핵심 UX | 유언장 위반 가능성이 있으면 주문 전 경고 |

---

## 핵심 피드백 루프

```
[1] 유언장 작성 (선택)
    평정심 있을 때 → 나만의 투자 원칙을 글로 씀
    "나는 +20% 급등 코인은 사지 않는다"
    "새벽 2시 이후엔 거래하지 않는다"
         ↓
[2] 매수 시
    유언장 집행 → 원칙 어기고 있는지 체크 (과정 평가)
    매수 기록   → 왜 지금 사는지 thesis 작성
         ↓
[3] 보유 중
    중간 진단   → 목표가/손절가/경과 기간 기반 상태 알림
                  유언장 건드리지 않음 (결과 미확정)
         ↓
[4] 매도 후
    최종 보고서 → 과정(원칙 지켰나) + 결과(수익/손실) 합산
    유언장 최적화 제안 → "이 패턴에서 3번 손실. 원칙 추가할까요?"
```

## 거래 전 유언장 검사 — 위험 조건 매핑

| 검사 항목 | 감지 기준 | 유언장 예시 | 서비스 반응 |
|----------|----------|------------|------------|
| 급등 후 추격 매수 | 최근 1시간 +15% 이상 | "급등 직후 시장가로 들어가지 않는다." | 주문 전 경고 |
| 프리마켓 갭상 매수 | 프리마켓 전일 종가 대비 +N% 이상 | "프리마켓 갭상 직후 매수하지 않는다." | 주문 보류 권고 |
| 손절 기준 없음 | 주문 전 손절가/손절률 미입력 | "손절 기준 없는 거래는 시작하지 않는다." | 손절 기준 입력 요구 |
| 전액 일괄 매수 | 투자 가능 금액 대비 높은 비중 일괄 | "확신해도 분할 진입한다." | 금액 축소/분할 제안 |
| 손실 만회 재진입 | 직전 손실 후 같은 종목 재매수 | "복수 매매를 하지 않는다." | 강행 사유 입력 |
| 시장가 충동 매수 | 급등 구간 + 시장가 주문 | "과열 구간에서는 시장가 주문을 피한다." | 지정가 전환 제안 |
| 반복 조회 후 주문 | 같은 종목 상세 화면 N회 이상 조회 후 주문 | "조급함으로 주문하지 않는다." | 쿨다운 제안 |

```
홈
├── 활성 거래 목록 (보유 중) + 매도 버튼

├── 유언장 조항 제안 + 승인/수정/버리기
└── 위반 조항 링크

유언장 관리
├── 조항 목록 (rule_type + params)
├── 조항별 위반 횟수 / 연관 손실
└── AI 최적화 제안 내역
```

## 구현 단계 (P1–P4)

### P1 — Broker + 척추 (백엔드 통합)

- **[필수]** `BrokerService` 인터페이스 정의
- **[필수]** `MockBroker`: 현재가(등락률 포함), 주문 전송, 체결 조회, 잔고 — 위반 강제 트리거 가능한 결정론적 데이터
- **[필수]** 워킹 스켈레톤 — 전 구간 stub 연결 (가짜로라도 전체가 돌아야 함)
- **[필수]** `POST /api/order/precheck` — 시장데이터 fetch → `checkOrder` 호출 → 위반 반환
- **[필수]** `POST /api/order/execute` — 강행/통과 → broker 주문 → 체결 → Trade/Order 기록
- **[필수]** Trade 생명주기 — 매수 체결 시 OPEN, 매도 시 CLOSED
- **[필수]** 매도 시 손익·보유시간 계산 → P3 `generateReport` 트리거
- **[필수]** 메인 브랜치 오너 / 통합
- [여유] `KisBroker` — 토큰 발급+캐시 + 읽기 전용 현재가만

### P2 — 프론트 3화면

- **[필수]** 주문 입력 화면 — 현재가·등락률 표시, 수량/주문유형/손절가/매수이유 입력
- **[필수]** precheck 호출 → 위반 있으면 낭독 모달
- **[필수]** 낭독 모달 — 위반 조항 문구 + 현재 상황 + 선택지 (미루기/지정가/금액축소/손절작성/강행)
- **[필수]** 강행 사유 입력 → execute 호출
- **[필수]** 보고서 화면 — 사망/생존 렌더 + 조항 제안 + 승인/수정/버리기 버튼
- **[필수]** 보유 거래 + 매도 버튼 (매도가 보고서 트리거)
- [여유] 조항 목록 (읽기 전용), 로딩/에러 상태 폴리시

### P3 — Gemini 복기

- **[필수]** `generateReport(trade)` — 손익 판정 → 사망진단서 or 생존보고서 분기
- **[필수]** 사망진단서: 손실 원인 + 위반 조항 + 위험 행동
- **[필수]** 생존보고서: 수익 원인 + 지킨 조항 + 재현할 습관
- **[필수]** 조항 제안 — 기존 rule_type + params 조정만 (자유텍스트 금지), JSON 구조화 출력
- **[필수]** Gemini 호출 + JSON 파싱 + 에러 핸들링 (파싱 실패 대비 fallback)
- mock 거래 JSON으로 독립 개발 → P1 안 끝나도 안 막힘

### P4 — 규칙 엔진 + 조항

- **[필수]** 룰 함수 2개: `chaseSurge`, `noStopLoss`
- **[필수]** `checkOrder(order, market, clauses)` 모듈 → P1에 납품
- **[필수]** 조항 시드 2~3개 하드코딩 (급등추격·손절없음)
- **[필수]** 제안 적용 로직 — P3 제안 승인 시 조항 params 갱신 → 다음 주문에서 즉시 반영 (데모 클라이맥스)
- [여유] 조항 추가/삭제 CRUD, 일찍 끝나면 프론트 지원
