# Mortis — 기획 설계서

> "가장 좋은 투자 조언자는 AI가 아니라, 감정이 없을 때의 나일 수 있습니다."

**해커톤:** SKYSH 2026  
**주제:** Theme 1 — Emotion (기술로 공포와 탐욕 다루기)  
**날짜:** 2026-06-28

---

## 한 줄 설명

손실 거래를 부검하고, 평정심 있을 때 쓴 투자 유언장이 감정적 매수를 막아주는 서비스.

---

## 핵심 루프

```
[1] 유언장 작성 (선택)
    평정심 있을 때 → 나만의 투자 원칙을 글로 씀
    "나는 +20% 급등 코인은 사지 않는다"
    "새벽 2시 이후엔 거래하지 않는다"
         ↓
[2] 매수 시
    유언장 집행 → 원칙 어기고 있는지 체크 (과정 평가)
    매수 기록   → 왜 지금 사는지 thesis 작성
         ↓
[3] 보유 중
    중간 진단   → 목표가/손절가/경과 기간 기반 상태 알림
                  유언장 건드리지 않음 (결과 미확정)
         ↓
[4] 매도 후
    최종 보고서 → 과정(원칙 지켰나) + 결과(수익/손실) 합산
    유언장 최적화 제안 → "이 패턴에서 3번 손실. 원칙 추가할까요?"
```

## 기능 목록 (장의사 테마)

| 기능명 | 설명 |
|--------|------|
| **투자 유언장** | 평정심일 때 작성하는 나만의 투자 원칙 |
| **유언장 집행** | 매수 시 관련 원칙 자동 표시 + 위반 경고 |
| **거래 기록** | 매수 이유, 목표가, 손절가 기록 |
| **거래 부검** | 매도 후 손실 원인 분석 리포트 (사망진단서) |
| **생존 보고서** | 매도 후 수익 거래 분석 |
| **사인 데이터베이스** | 반복되는 손실 패턴 누적 |
| **망자의 경고** | 같은 실수 감지 시 경고 발동 |
| **거래 공동묘지** | 전체 손실 거래 히스토리 |
| **유언장 최적화** | 보고서 기반 AI 원칙 개선 제안 |

## 보고서 예시

### 거래 부검 (손실)
```
사망진단서

자산: KRW-BTC
진입: 2026-06-15 02:13 / 143,200,000원
청산: 2026-06-17 11:40 / 128,500,000원
손실: -10.3%

사망 원인: 급등 직후 추격 매수

보조 원인:
- 진입 3분 전 커뮤니티 글 확인
- 새벽 시간대 거래 (유언장 위반)
- 손절가 미설정

예방 가능성: 높음
유언장 위반 항목: 2건

유언장에 추가할까요?
→ "커뮤니티 글 확인 후 10분 내 진입하지 않는다" [추가] [건너뜀]
```

### 중간 진단 (보유 중)
```
중환자실 모니터링

자산: KRW-ETH  
보유 기간: 12일
현재 수익률: -8.2%

손절가(-15%) 까지 남은 거리: 7.8%
목표가(+20%) 까지 남은 거리: 28.2%

상태: 위험 신호 감지
→ 지금 팔면 어떤 보고서가 나올지 미리 보기
```

## 화면 구성

```
홈
├── 활성 거래 목록 (보유 중)
├── 최근 부검 보고서
└── 유언장 요약

매수 화면
├── 유언장 집행 (해당 원칙 표시)
├── 매수 기록 작성 (왜 사는가)
└── 목표가 / 손절가 설정

보고서 화면
├── 사망진단서 / 생존 보고서
├── 유언장 최적화 제안
└── 사인 데이터베이스 업데이트

유언장 관리
├── 원칙 목록
├── 원칙별 위반 횟수 / 연관 손실
└── AI 최적화 제안 내역

거래 공동묘지
└── 전체 손실 거래 타임라인
```

## 기술 스택

| 역할 | 도구 |
|------|------|
| 프론트 + API 서버 | Next.js 15 (App Router) |
| 데이터베이스 | Supabase (PostgreSQL) |
| AI 판정 + 코칭 | Claude API (claude-sonnet-4-6) |
| 거래 데이터 | 업비트 REST API |
| 스타일 | Tailwind CSS |

---

## 보안 설계 (해커톤 요구사항)

- 업비트 API Key → 브라우저 localStorage에만 저장
- 서버 DB에 저장 금지
- API 호출 시 클라이언트가 Authorization 헤더로 전달
- Next.js API Route는 중계만 (키 로깅 없음)

## 데이터 모델

```
Trade
  id, asset, side(buy/sell), price, quantity
  executed_at, upbit_uuid

TradeNote (매수 시 기록)
  trade_id, thesis(text)
  target_price, stop_price
  will_violations[] (유언장 위반 항목)

Evaluation (매도 후)
  trade_id
  direction_correct(bool), target_hit(bool)
  process_score(1-5), outcome_score(1-5)
  ai_report(text), ai_suggestions[]
  user_confirmed_at

Will (유언장)
  id, rule(text), created_at
  violation_count, related_loss_total

WillSuggestion (유언장 최적화 제안)
  evaluation_id, suggested_rule(text)
  accepted(bool)
```

## 업비트 API 활용

- `GET /v1/orders` — 체결 내역 자동 연동
- `GET /v1/candles` — 가격 데이터 (목표가/손절가 도달 체크)
- `GET /v1/ticker` — 실시간 현재가 (중간 진단)

## MVP 범위 (해커톤)

**포함**
- 유언장 작성 / 수정
- 매수 시 유언장 집행 + 기록
- 매도 후 부검 보고서 (AI)
- 유언장 최적화 제안

**V2로 미룸**
- 거래 공동묘지 시각화
- 사인 데이터베이스 차트
- MCP 연동
